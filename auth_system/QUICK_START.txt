ACCESS APPLICATION:
──────────────────

───────────────

docker-compose down          # Stop all services

🎯 TESTING THE SECURITY
═══════════════════════════════════════════════════════

TEST CSRF PROTECTION:
────────────────────
1. Try to register with devtools network tab open
2. Copy the form in browser console
3. Remove csrf_token field and submit
4. Should get: "Security token mismatch" error

TEST RATE LIMITING:
──────────────────
1. Try to login 6 times with wrong password
2. Should see: "Too many login attempts" message
3. Wait 15 minutes (or restart to test with cleared session)

TEST INPUT VALIDATION:
─────────────────────
1. Try username < 3 characters
2. Try invalid email format
3. Try password < 8 characters
4. Try passwords that don't match
5. All should show appropriate validation errors

TEST SQL INJECTION:
──────────────────
1. Try login with email: admin' OR '1'='1
2. Should get: "Invalid email or password"
3. Check database - no injection occurred

🚨 TROUBLESHOOTING
═══════════════════════════════════════════════════════

Container won't start:
→ Check logs: docker-compose logs app
→ Verify .env file exists
→ Check disk space: docker system df

Can't connect to database:
→ Ensure MySQL container is healthy: docker-compose ps
→ Wait 10 seconds for MySQL to be ready
→ Check DATABASE credentials in .env

Port 80 already in use:
→ Change in docker-compose.yml: "8080:80"
→ Access at: http://localhost:8080

SSL certificate errors (production):
→ Generate: certbot certonly --standalone -d your-domain.com
→ Add cert path to nginx.conf and uncomment HTTPS section
→ Set FORCE_HTTPS=true in .env
